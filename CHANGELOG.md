## [1.0.0] - 2025-12-23

### 新增功能 ✨

#### 核心系統
- **完整的多代理辯證系統**：實作 Orchestrator、三個 Perspective Agents、Critic 五個專業角色
- **六階段辯證流程**：Phase 0-6 完整實作，包含需求分析、方案生成、批判審查、共識檢查等
- **並行代理執行**：Phase 1 和 Phase 3 支援三個代理真正並行處理，大幅提升效率
- **智能角度配置**：根據需求類型自動選擇最適合的思考角度（架構設計、功能開發、效能優化等）

#### 評估與共識機制
- **量化評估系統**：可行性、效益、風險控制三維度評分，每項 10 分，總分 30 分
- **智能共識檢查**：支援多數同意（≥2 個 Agent）和明顯領先（分數差距 ≥8）兩種共識模式
- **Critic 最終裁決**：當達到最大輪數仍無共識時，由 Critic 進行最終判決

#### 使用者互動
- **多種互動選項**：繼續辯證、採納方案、介入調整、重設角度等
- **階段性決策支援**：支援複雜問題分階段進行辯證分析
- **混合方案策略**：當各方案都有優點時，支援提出混合解決方案

#### 自動觸發機制
- **關鍵字智能檢測**：自動識別「辯證」、「多角度分析」、「方案比較」等觸發詞
- **情境觸發**：識別「設計系統 + 考慮因素」、「選擇技術 + 評估標準」等情境
- **參數化支援**：支援 `--max-rounds N` 和 `--perspectives "角度1,角度2,角度3"` 參數

#### MCP 工具整合
- **Sequential Thinking**：提供結構化思考和推理能力
- **Context7**：存取最新的程式庫文檔和 API 參考
- **Serena**：程式碼分析、檔案操作和專案管理工具

#### 輸出與報告
- **結構化報告生成**：包含最終方案、評分明細、關鍵洞察、實施建議
- **完整辯論記錄**：保存所有輪次的方案、挑戰、回應和評分
- **可摺疊詳細記錄**：支援點擊展開查看完整辯論過程

### 技術特性 🔧

#### 系統架構
- **Kiro Subagent 整合**：使用 Kiro 原生的 subagent 系統執行
- **並行處理優化**：透過同一個 function_calls block 實現真正的並行執行
- **模組化設計**：清晰的角色分離和階段劃分，便於維護和擴展

#### 效能優化
- **並行方案生成**：Phase 1 三個 Agent 同時生成初始方案
- **並行回應處理**：Phase 3 三個 Agent 同時回應 Critic 挑戰
- **智能共識檢查**：避免不必要的辯證輪次，提升決策效率

#### 可擴展性
- **角度配置靈活性**：支援自動配置和手動指定思考角度
- **評估標準可調**：支援在需求描述中指定特殊評估標準
- **輪數控制**：支援設定最大辯證輪數，適應不同複雜度需求
